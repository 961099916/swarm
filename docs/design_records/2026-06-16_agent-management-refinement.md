# LLD 底层设计文档: 智能体管理后台前端页面实现方案

本方案旨在为系统管控后台新增“全局智能体管理”前端页面，使用户能够通过管理后台直观地查阅、微调和下线全平台所有的智能体，彻底打通管理端的全局管控链路。

---

## 1. 架构定位与页面关系
- **路由及页面挂载**：
  - 新建页面：agents.vue（作为后台智能体集中管控控制台）。
  - 入口挂载：在 dashboard.vue 的“系统级特权控制”列表中新增“智能体管理”卡片。
  - 路由注册：在 pages.json 中追加新路由配置 `/pages/admin/agents`。

---

## 2. 页面交互设计与核心契约 (Contract Design)

### 2.1 交互设计
- **全局卡片列表**：呈现系统内的所有智能体。卡片上清晰标注 `内置预设` 或是 `自定义` 标签，对于自定义智能体，需额外展示 `创建者 ID (UserId)`。
- **全局参数微调**：点击卡片的“✏️ 调整指令”调起磨砂玻璃弹窗，回显参数，允许直接修改其 Prompt、头像或工具链绑定（调用 `PUT /api/v1/admin/agents/update`）。
- **全局强制下线**：点击“🗑️ 强制下线”弹出二次确认，调用 `DELETE /api/v1/admin/agents/delete` 进行下线。

---

## 3. 防御设计与边界校验 (Defensive Design)

1. **路由阻断防护**：
   - 在页面 `onShow` 阶段校验当前登录用户的身份 `userStore.userRole !== 'ADMIN'`，如果未授权则直接强制拦截并退回个人中心，防止直接扫码/URL 注入越权访问。
2. **预设智能体操作封禁**：
   - 列表中对于系统预设（`isPreset === true`）的智能体，前端交互上灰度化并物理隐藏“✏️ 调整指令”和“🗑️ 强制下线”按钮，从人机交互（HCI）层面阻止越权尝试。
