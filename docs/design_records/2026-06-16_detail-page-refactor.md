# ADR: 协同终端详情页(detail.vue)解耦重构设计决策

## 上下文
`detail.vue` 承担了协同终端的详情查看和实时日志跟踪，由于其包含大量的复杂交互、轮询逻辑和巨量 SCSS 样式代码，文件长度达 820+ 行，严重违反了阿里巴巴开发规约和单一职责设计原则。

## 决策
1. **纯逻辑钩子抽离**：抽取 `useTaskLogs.ts`，利用 Vue3 组合式 API 的力量将轮询、数据源清理合并及滚动控制高内聚隔离出来。
2. **组件分治解耦**：
   - 提取 `StatusCard.vue` 渲染基本状态和元数据看板；
   - 提取 `AIChatDetailCard.vue` 处理 AI 消息模型交互和手风琴内容细节；
   - 提取 `TerminalConsole.vue` 组合命令行、文本行和 `AIChatDetailCard.vue` 的交互渲染。
3. **主页面极致精简**：重构 `detail.vue` 后，其行数将控制在 100 行内，仅充当页面壳和状态的分发者。

## 影响
- 大幅提高了代码的可读性和调试效率。
- 符合阿里巴巴前端规范。
- 样式划分更清晰，不会发生不必要的样式继承和污染。
