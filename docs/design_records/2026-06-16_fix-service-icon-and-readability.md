# Architecture Decision Record (ADR) - 修复服务图标与全局配色可读性重塑

## 1. 背景与上下文

在 Swarm 微信小程序近期版本中，出现了两处较为严重的 UI 渲染及视觉体验问题：
1. **`t-icon-service` 像素图标显示失效**：
   - 智能体工作台及个人中心里的“自定义智能体工作台”菜单图标显示为空白。经排查，其遮罩背景的 SVG Base64 编码中含有非法的越界属性 `<rect x='1' y='66' width='1' height='1'/>`，超过了 8x8 的 `viewBox`，导致 SVG 引擎抛出解析异常，渲染终止。
2. **全局配色在深色背景下的可读性（Readability）极差**：
   - 原设计将小程序全局 `page` 级的前景色字色设置为深原木褐色（`#3d2511`），而全局底色（`--bg-root`）为深木褐色（`#2d1e13`）。
   - 这使得所有没有被羊皮纸卡片（`.card`，其底色是米黄色的 `#f5e3ca`）包裹的文本（例如列表标题、分组标题、以及无背景的内容）在深色背景下显示为极暗的褐色，形成了“深底深字”的现象，对比度几乎为零，严重影响用户阅读。

---

## 2. 决策与设计方案

为保证代码的整洁度及全栈架构的高工程化表现，决定采取以下级联式样式修补方案：

1. **修正 `t-icon-service` 的 SVG Base64 编码**：
   - 将越界的 `y='66'` 修正为合法的 `y='6'`。
   - 重新计算其 SVG 的 Base64 编码串并更新至全局样式文件。
   
2. **重塑全局色彩与对比度继承链**：
   - 重构 `app.wxss` 中全局 `page` 节点样式，将默认文字颜色（`color`）置为高亮对比的亮米黄色 `--text-light`（即 `#f5e3ca` 或 `#dfb180`）。
   - 在 `.card`、`.card-premium`、`.card-stat` 等卡片级公共选择器上强制显式声明文字颜色 `color: var(--text, #3d2511) !important`，确保当文字被羊皮纸米黄卡片承载时，对比度依然为深色对亮底。
   - 如此设计能够保证：
     - 在深木色背景上的直接文字，呈现明亮淡黄色，保证极佳的可读性。
     - 在卡片容器内部的文字，呈现深黑褐色，提供极度舒适且清晰的阅读体验。
     - 极大地降低了针对每一个细节 text class 挨个修改的工程碎片化弊病。

## 4. 验证方式

- 检查静态样式，通过 regex 和 decode 工具确保 SVG 语法 100% 正确。
- 动态编译核对：
  - “自定义智能体工作台”图标（`.t-icon-service`）复现并支持跟随色彩继承。
  - 暗色背景上的段落标题（如 “我创造的智能体” 等）可清晰正常地读取。
  - 卡片内部表单文字高对比呈现。
