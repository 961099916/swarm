# ADR: 浅色模式字体与配色修复设计决策

## 上下文
Swarm 小程序前端在暗色模式下拥有极佳的视觉表现力，但在切换为浅色模式时，由于部分核心变量（青色、金色、紫色等）缺失，且某些页面的文本颜色（如终端日志、任务卡片）直接使用固定的浅白灰值，导致浅色模式下文字对比度低、高光半透明白底模糊等问题，严重影响无障碍阅读体验。

## 决策
1. **补齐基础变量映射**：在 `themeStore.ts` 的 `LIGHT_THEME_VARS` 中，精确对齐 `App.vue` 中的暗色主题变量（补齐 `--color-cyan`、`--color-gold`、`--color-purple` 等及其衍生透明度和高亮颜色变量）。
2. **优化文字在亮色模式下的表现**：在亮色模式下提升青色与金色的字色深度，提升在纯白底/浅灰色底下的可读性（如 `--color-cyan-light` 变为深青 `#08979C`）。
3. **增加 `.theme-light` CSS 权重覆写**：
   - 重构 `detail.vue` 终端框背景，在浅色模式下采用低饱和度的高雅浅灰蓝（`#F0F2F7`），同时将命令行、提示词、时间戳及日志文字在亮色模式下全部转换至深冷灰色系。
   - 优化 `task/index.vue` 及 `dashboard.vue` 等页面的半透明底色和高亮文本，使其完美适应亮色背景。

## 影响
- 无任何业务逻辑破坏，仅调整前端 CSS 变量映射与样式覆盖。
- 提升了亮色模式的易用性与一致性。
