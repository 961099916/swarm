# ADR: 原生微信小程序 + TDesign + TailwindCSS 全面重塑决策

## 上下文
原本的前端基于 UniApp 跨端框架开发，存在因构建工具繁重、编译警告多、且对于三方组件样式穿透隔离层繁琐等痛点。为了追求极致的原生小程序运行速度与腾讯官方 TDesign 的高端大厂质感，决定将项目从 UniApp 全面重构迁移为微信小程序原生工程。

## 决策
1. **剥离跨平台框架**：彻底废除 UniApp、Vue3 与 Vite 编译管道，回归微信小程序标准的 `WXML`、`WXSS`、`TS` 与 `JSON` 原生四元组开发模式。
2. **状态与网络管理重构**：状态集中保存在全局 `App.globalData` 内。网络通信基于 `wx.request` 独立封装，支持一键拦截并携带全局 TraceID 和 Auth Token。
3. **融合 TDesign Miniprogram**：全面通过 npm 引入腾讯官方微信小程序原生组件库，并在 `.json` 中注册引入。
4. **接入预编译原子类**：在全局引入微信小程序兼容的预编译 `tailwind.wxss` 原子样式库，解决原生小程序自定义 CSS 样式冗余和难以维护的问题。

## 影响
- 获得了最快的真机启动性能与最轻的包体积。
- 获得了最纯粹的大厂官方交互样式，且不依赖任何 Node/Vite 复杂的后处理编译插件，架构极为稳健、寿命长久。
