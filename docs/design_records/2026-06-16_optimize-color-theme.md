# Architecture Decision Record (ADR)

## 整体配色优化与阅读性提升决策

- **日期**: 2026-06-16
- **状态**: 已批准 (Approved)
- **关联组件**: 全局像素风格设计体系 (Retro-Pixel Theme)
- **决策人**: Antigravity (首席全栈架构师)

---

### 1. 问题背景
Swarm 小程序采用 16-bit 复古像素风设计。此前存在部分色彩定义可读性过低的问题：
1. 主色黄铜金 (`--brand`: `#d39c3f`) 的辅助色 `--brand-subtle` 被定为固定的木黄色 (`#dfb180`)，两者亮度相差极微弱（对比度不足 **1.1:1**）。当主色调图标（如“自定义智能体工作台”）放置在其底座背景上时，图标与背景严重“粘连”，几乎完全隐形。
2. 默认明亮卡片背景上的文字颜色（`--text-secondary` 和 `--text-muted`）由于红褐色的明度偏高，导致长文本或小字号提示在羊皮纸质卡片上的辨识度不够爽朗，容易造成视觉疲劳。

---

### 2. 决策与优化细节
为了确保在保留星露谷像素木质画风的基础上拉大明暗对比度，提升阅读清晰度，我们对 `frontend/app.wxss` 中的全局变量进行了精细化调整：

#### 2.1 默认主题变量调优
- **`--brand-subtle`**: `#dfb180` $\rightarrow$ `rgba(211, 156, 63, 0.12)`
  - **决策原因**: 从固定深木黄改为 12% 不透明度的主色调。这产生了一个极淡的金色底图背景，大幅度将图标与底座的对比度拉大，彻底解决了图形不可见的问题。
- **`--text`**: `#3d2511` $\rightarrow$ `#2b190b` (加深 26% 亮度)
  - **决策原因**: 像素风字体边缘通常有粗糙锯齿，加深原木文字的色值能让卡片正文阅读起来更具穿透力。
- **`--text-secondary`**: `#57371d` $\rightarrow$ `#3f2410`
- **`--text-muted`**: `#854c30` $\rightarrow$ `#60351b`
  - **决策原因**: 将小字号提示文字从高亮偏红调深为清晰的泥土褐色，极大提高了卡片辅助信息的识别度。
- **`--border-default`**: `#57371d` $\rightarrow$ `#4a2c16`
  - **决策原因**: 卡片边缘线条微调加深，进一步凸显 16-bit 像素框线的粗矿立体浮雕感。

#### 2.2 亮色主题覆盖变量同步调优 (`.theme-light`)
- **`--text`**: `#2d1e13` $\rightarrow$ `#21140a`
- **`--text-secondary`**: `#3d2511` $\rightarrow$ `#2d1a0d`
- **`--text-muted`**: `#57371d` $\rightarrow$ `#4a2c16`

---

### 3. 架构影响与向下兼容
- **视觉无缝升级**: 纯变量色值调优，对已有 DOM 结构无任何侵入。
- **极佳的性能与兼容**: 直接利用浏览器/小程序的 CSS 变量继承树机制响应主题切换，不引入任何额外的重绘 JS 逻辑。
