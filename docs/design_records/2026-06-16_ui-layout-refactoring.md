# ADR: 微信原生小程序 UI 布局重构与样式规范设计记录

## 1. 架构定位与解耦设计
本项目针对微信原生小程序中的所有 UI 布局进行深度清洗与美化，彻底清理在微信原生 WXSS 编译器中会触发词法错误的 Tailwind 原子类名（如包含中括号 `[...]`、点号 `.`、斜杠 `/` 以及需要反斜杠转义的选择器），以提升编译稳定性和首屏渲染质量。
- **UI 呈现与交互层**：继续使用 `TDesign Miniprogram` 作为基础交互组件，结合经过清洗的常规类名控制结构和自适应排版。
- **样式解耦与规范定义**：
  - 核心排版不再直接依赖带有特殊符号的 Tailwind 类名。
  - 常用的微调间距、特殊比例宽度、字体大小等，统一抽离到各页面私有的 `.wxss` 文件中，通过标准的 CSS 变量和常规类名（例如 `.gap-1-5`、`.w-3-5` 等）进行安全定义。
  - 精细定义亮/暗色双主题下的色彩渐变、毛玻璃磨砂（Glassmorphism）与卡片阴影，确保在双色模式下能够平滑转换，实现企业级的高端视觉交互。

---

## 2. 核心契约与命名约束

### 2.1 CSS 命名规范契约
为了防止微信小程序原生 WXSS 编译器出现编译错误，本项目严禁在 WXML 和 WXSS 中使用下列格式的类名：
- **禁止包含中括号**：如 `text-[16rpx]`、`max-w-[200rpx]`、`w-[580rpx]`。
- **禁止包含未转义的点号/斜杠**：如 `gap-1.5`、`w-3/5`。
- **允许的常规替代命名规范**：
  - 点号用 `-` 代替：如 `gap-1.5` -> `gap-1-5`；`h-1.5` -> `h-1-5`。
  - 斜杠用 `-` 代替：如 `w-3/5` -> `w-3-5`。
  - 特殊宽度直接在对应 `.wxss` 中使用常规类名声明：如 `.w-580rpx { width: 580rpx; }`。

---

## 3. 控制流转与设计模式

### 3.1 主题穿透控制流
所有页面布局的根容器必须绑定 `theme-{{theme}}`。在 `app.wxss` 中声明的 CSS 变量会通过类名继承层层穿透，卡片背景、文字颜色、边框等均使用 `var(--bg-surface)`、`var(--color-text-primary)` 等变量实现秒级零抖动切换。

---

## 4. 防御设计 (异常处理与容错)
1. **语法高危防护（编译器级兜底）**：在重构完 WXML 文件的类名后，在本地对 `tailwind.wxss` 文件进行彻底检索，剔除并重写所有含有 `\`、`[`、`]`、`/`、`.` 的选择器定义，从源头切断微信 WXSS 编译器的报错隐患。
2. **极窄屏幕布局坍塌防护**：所有的 Bento 网络布局与卡片弹性盒（Flexbox）容器均配置 `flex-wrap: wrap` 或 `box-sizing: border-box`，确保在 320px 宽度的超旧型设备上文字不溢出、布局不崩塌。
3. **TDesign 弹出层阻断穿透**：针对创建智能体等弹窗（`t-popup`），在 WXML 中加入滑动拦截，防止弹窗滚动时导致底层背景视差穿透，影响用户体验。
