# 架构决策记录 (ADR)：修复工作流收尾归档结果优先级漏洞

* **状态**：已批准 (Approved)
* **日期**：2026-06-17
* **作者**：Antigravity (首席全栈架构师)
* **上下文 (Context)**：在协同任务执行完毕时，主控协调官通过 `FINISH` 决策输出了自然语言总结，但引擎将其错误覆写为了工具调用最后一次留在对话历史中的原始 JSON 串，导致最终输出结果丧失了可读性。

---

## 决策 (Decision)

在 [workflow.ts](file:///Users/zhangjiahao/IdeaProjects/swarm/backend/workers/workflow/src/workflow.ts) 的 `finalize-orchestration` 步骤中，重构最终结果摘要的富集逻辑，采取**主控决策摘要优先**，**历史详情降级兜底**的策略：

1. 如果主控的 `finalSummary` 不为空，且不是默认占位符 `"所有智能体协同动作完成"`，则将其设为最终向 D1 写入的任务报告。
2. 否则，将降级搜索提取最后一次 Agent 回复或工具返回数据作为兜底展现。

---

## 评估与后果 (Consequences)

* **优点**：
  - 完全理顺了 Supervisor 架构的终结语义，保障最终输出的自然语言易读性。
  - 维持架构高内聚低耦合，仅涉及工作流引擎终结节点的流转逻辑变更，不改变数据库契约与前端 API。
* **影响**：需确保没有破坏多智能体协同非天气任务的最终结果提取展示。
