# 架构决策记录 - 项目无用文件与文档清理

- **状态**: 已接受
- **日期**: 2026-06-20
- **决策者**: 反重力开发助手 & 架构评审官

## 背景
随着 Swarm 项目的演进和工程化重构，项目中引入了大量开源合规性文档以及多达 65 篇的历史架构决策记录。这些文档对于当前的纯私有部署与企业级全栈开发造成了以下负面影响：
1. **IDE 索引冗余**: 在进行全局文本或文件搜索时，历史 ADR 文档会产生大量无关干扰匹配。
2. **工作区杂乱**: 冗余文件使得项目结构不够精简，增加了维护心智成本。
3. **开源模板废弃**: 作为商业/私有性质的企业级多 Agent 平台，开源行为准则、贡献指南等合规模板不再适用。

## 决策
决定对项目中的冗余文档进行物理清理。具体清理策略如下：
1. **删除开源合规相关文件**: 清理根目录下的 `CODE_OF_CONDUCT.md`、`CONTRIBUTING.md`、`SECURITY.md`、`CHANGELOG.md`、`LICENSE`。
2. **删除 docs 非核心文档**: 清理 `docs/ADRs.md`、`docs/development-process.md` 和 `docs/secrets-management.md`。
3. **彻底清空历史架构决策记录**: 清空 `docs/design_records/` 下除本篇归档外的全部 65 篇历史记录，以净化 IDE 搜索空间。
4. **保留本地规范**: 维持根目录下 `agent.md` 和 `SCRIPTS_README.md`，保证 Agent 在开发过程中的代码洁癖规范可用。
5. **统一提交**: 在清理完成后，将变动与先前被物理删除的旧前端小程序文件一同提交至 Git 仓库。

## 后果
- **正面**: 
  - 大幅精简代码库，减少 IDE 的文件搜索与文本匹配索引负担。
  - 工作区结构清爽，只包含核心功能与当前维护的业务代码。
- **负面**: 
  - 历史设计演进过程将无法直接在本地 markdown 文件中查看（但可以通过 Git Commit 历史进行追溯）。
- **迁移与后续**: 
  - 本次清理产生的变动将在清理和验证构建无误后，统一提交。
