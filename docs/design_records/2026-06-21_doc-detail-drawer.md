# 架构决策记录 (ADR) - 知识库聊天页面点击参考文档弹出详情抽屉

- **状态**: 批准执行 (Approved)
- **日期**: 2026-06-21
- **作者**: Antigravity

## 1. 上下文与问题描述
在知识库智能问答页面（`packageKnowledge/chat`）中，AI 的回答下方会渲染相关的“参考文档”标签（表示该回答基于哪些文档生成的）。用户希望在点击这些标签时，能够拉起侧边抽屉展示该文档的全文详情内容，以此免去用户切换到其它页面去查找原文的繁琐流程。

## 2. 决策与设计方案
我们决定在小程序端引入侧拉抽屉（TDesign `t-popup` 配合右侧弹出模式），并在抽屉内使用 `mp-html` 解析渲染富文本。

### 2.1 懒加载与缓存机制
为保证首屏加载性能与流量节省，我们不预先加载全部文档的原文内容。
- 当用户点击具体文档标签时，程序首先检查本地 `this.data.documents` 是否为空。
- 若为空，则调用后端 `GET /api/v1/kb/document/list?kbId=xxx` 获取所有文档原文（含 `rawContent`）并缓存在内存中。
- 若已存在，则直接在本地进行检索与渲染。

### 2.2 标题匹配策略
标签展示的文档名格式（如 `MySQL基础.md` 或 `MySQL`）可能会与数据库中存储的 `title` 有微小差异（如大小写、首尾空格或后缀的有无）。
我们采用正则/字符串清理策略（去除后缀、转为小写、剔除空格）进行模糊与匹配，确保点击时的高命中率。

### 2.3 状态机定义
- `showDocPopup` (`boolean`): 是否开启详情侧边抽屉。
- `selectedDocTitle` (`string`): 选中的文档标题。
- `selectedDocContent` (`string`): 选中的文档 HTML 格式正文。
- `docDetailLoading` (`boolean`): 正在拉取原文时的 Loading 状态。

## 3. 防御设计
- **接口防抖与限流**：网络请求期间，设置 `docDetailLoading: true`，在此期间多次点击参考文档标签将被拦截，避免并发重复发起请求。
- **解析错误兜底**：如果文档没有 `rawContent`，渲染“*暂无文档内容*”。
- **全链路追踪**：请求 Header 携带 `X-Trace-Id`，方便在服务端日志系统中检索定位。
