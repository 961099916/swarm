# 架构决策记录 (ADR) - 自定义流程可视化 Canvas 预览与详情页执行态连线高亮及卡片状态追踪

- **状态**: 批准执行 (Approved)
- **日期**: 2026-06-21
- **作者**: Antigravity

## 1. 上下文与问题描述
在支持自定义工作流静态编排的场景下，用户由于只能在小程序通过文字卡片列表和下拉框定义连接关系，缺乏空间拓扑的直观预览，很难在脑海中勾勒出“流程的全局图景”与“条件分支走向”。
为了在手机窄屏端无缝提供高颜值的图形化预览，我们设计开发了零外部依赖的 Canvas 流程预览器。
同时，用户在任务运行后需要知道“下一步条件判定”的实际执行走向，这要求在详情页将静态的拓扑结构与动态的运行日志、步骤转移状态完美结合，并在界面中高亮显示“已被走通的路径”。

## 2. 决策与设计方案

### 2.1 零外部依赖的 Canvas 2D 层级排版器 (FlowViewer)
在小程序端，我们采用原生的 HTML5 Canvas 2D 引擎：
- **层级布局计算 (Hierarchical Layout)**：在绘图前，基于 BFS 遍历确定步骤节点层级（Level）与同层对称横向偏移（X-Offset），以此决定每个步骤在画布中的 (X, Y) 坐标。
- **三次贝塞尔曲线 (Bezier Curve)**：对于节点跳转，使用 `bezierCurveTo` 从父节点输出底端画至子节点输入顶端，呈现平滑自然的线条交互。
- **条件气泡 (Condition Bubbles)**：对于 condition 分支节点的输出端连线，在贝塞尔线段的精确中点，绘制圆角背景小气泡，写入该分支对应的判定词 `matchValue`（如 `refund / negative`），达成条件可视化。

### 2.2 LLM Classifier 引导句式卡片 (Guided Condition Branch)
条件节点的判定完全基于大模型（LLM）的单词分类器能力。为了让这一条件在配置层高度可预测：
- **引导卡片**：小程序配置页卡片上设计为“句式引导输入表单”——*“当 AI 判定为 `[判定词]` 时 ➔ 跳转步骤 `[下拉选择目标]`”*。
- **强约束 Prompt**：执行状态机运行至该节点时，自动组装包含分支 `matchValue` 列表的限定 Prompt，强制大模型只能输出列表词中的一个，引擎基于文本比对完成跳转。

### 2.3 详情页执行态连线高亮 (Active Edge Highlights)
在任务详情页，除了对步骤节点进行状态标色（成功 -> 绿，运行中 -> 品牌色，排队 -> 灰，失败 -> 红），我们将对已实际发生转移的“连线”进行高亮染色：
- **跳转链路追踪**：页面解析日志时，通过遇到 `[Step-START]` 与 `[Step-BRANCH]`（命中分支或默认兜底）的跳转模式，解析出发生跳转的连线对 `fromStepId -> toStepId`，并存入 `activeEdges`。
- **染色绘制**：对于已被走通/激活的连线与条件气泡，Canvas 中以高亮绿（`#4CAF50`）或品牌色（`#FF6B6B`）且加粗渲染；未走过的分支连线则渲染为半透明淡灰以作弱化。

### 2.4 详情页步骤卡片判定结果展示 (Show Decision Results in Steps Card)
为了文字级确证跳转逻辑：
- 在解析日志时，将命中分支的文本（如 `“命中分支: 愤怒退款”`）写入该 `condition` 节点的 `branchResult` 字段。
- 时间轴列表中，在对应步骤下方以精美气泡小卡片展示其 `branchResult`。

## 3. 防御设计
- **Canvas 不存在防报错**：若 SelectorQuery 未能正确获取到 Canvas（例如渲染未完成或非工作流任务），逻辑应直接 return 阻断，不产生任何 JS 异常。
- **悬空节点对齐**：对于没有任何后续指向的非终点节点，自动分配至悬挂底层，防止画出视区外。
- **死循环限额**：限制图最大跳转次数为 20 次，超出立即熔断并置为 `FAILED`，保护 Token 算力。
